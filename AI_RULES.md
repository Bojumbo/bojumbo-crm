# AI PROJECT RULES: Bojumbo CRM

## 🎯 ГОЛОВНА ЦІЛЬ
Побудова CRM-системи на базі Laravel 12 з архітектурою "Numeric Field-First" та інтерфейсом у стилі Notion (Dark/Light).

---

## 🏗 1. АРХІТЕКТУРНЕ ЯДРО: "Numeric Field-First"
Кожне поле системи має унікальний статичний цифровий ID. Це фундамент для генерації документів та логування змін.

### Діапазони ID:
- **1000-1999**: Контрагенти (Counterparties)
- **2000-2999**: Угоди (Deals)
- **3000-3999**: Товари (Products)
- **10000+**: Кастомні поля користувача

### Словник полів (`fields_dictionary` або `fields_metadata`):
Таблиця, що зберігає мапінг: `ID -> Назва -> Тип даних -> Entity`.

---

## 🎨 2. ДИЗАЙН-СИСТЕМА (NOTION STYLE)
Суворе дотримання естетики Notion.

### Ключові параметри:
- **Шрифт:** `Inter` (14px base).
- **Сітка:** Крок 4px.
- **Радіус:** `4px` (стандарт), `3px` (кнопки), `6px` (модалки).
- **Кольори (Dark Mode):** 
  - Canvas: `#191919`
  - Sidebar: `#202020`
  - Text Primary: `rgba(255, 255, 255, 0.9)`
  - Text Gray: `rgba(255, 255, 255, 0.45)`
  - Border: `rgba(255, 255, 255, 0.09)`
- **Кольори (Light Mode):**
  - Canvas: `#FFFFFF`
  - Sidebar: `#FBFBFA`
  - Text Primary: `#37352F`
  - Border: `rgba(55, 53, 47, 0.09)`

### UI елементи:
- Таблиці: Тільки горизонтальні лінії.
- Inputs: Без рамок у спокої, легкий фокус.
- Іконки: Lucide (2px stroke).

---

## 💻 3. ТЕХНІЧНИЙ СТЕК
- **Backend:** Laravel 12 (PHP 8.4).
- **Frontend:** Tailwind CSS v4, Alpine.js, Blade.
- **Database:** MySQL/PostgreSQL + Redis.
- **Docs:** Google Docs API (інтеграція через `{{ID}}`).

---

## 🛠 4. ПРАВИЛА КОДУВАННЯ ДЛЯ AI
1. **Типізація:** Завжди використовувати строгу типізацію PHP (return types, property types).
2. **Локалізація:** Використовувати `__('key')` для всіх текстів. Підтримка RTL (Hebrew).
3. **Логування:** Кожна зміна поля сутності має записуватись в `activity_log` через його `static_id`.
4. **Контролери:** Тонкі контролери, логіка в Action-класах або Services.
5. **Tailwind:** Використовувати логічні властивості (`ps-`, `pe-`, `ms-`, `me-`) для підтримки LTR/RTL одним кодом.
6. **Верифікація змін:** Після кожного редагування, створення або будь-якої іншої дії з файлами проекту, AI має ще раз відкрити даний файл, перевірити чи застосувались зміни. Якщо зміни не застосувались — надати ПОВНИЙ змінений код файлу для ручної вставки користувачем.
